01 · О проекте

Концепция

Карьерная экосистема нового поколения

«Трамплин» — это интерактивная карьерная платформа, объединяющая студентов, выпускников и работодателей IT-индустрии. В отличие от классических job-бордов, Трамплин создаёт полноценную экосистему: здесь не просто ищут работу, а строят карьеру с нуля — находят менторов, участвуют в мероприятиях компаний и получают предложения о стажировках.

Проблема

Существующие решения слишком формальны или не учитывают специфику поиска первой работы и стажировки для студентов.

Решение

Единая платформа с картой возможностей, менторскими программами, карьерными мероприятиями и нетворкингом между соискателями.

Ценность

Студенты строят профиль, накапливают контакты и получают рекомендации — задолго до окончания вуза.

Личный кабинет

Соискатель · Построй карьеру

Личный кабинет соискателя — это центр управления карьерой. Здесь студент формирует профессиональный профиль, отслеживает отклики, управляет контактами и находит возможности.

Профиль

ФИО, вуз, курс, навыки-теги, резюме (PDF), репозитории GitHub, описание «О себе», аватар

Вкладки: вакансии и компании. Сохранение в localStorage для неавторизованных, синхронизация после входа

06 · Стек технологий · Frontend

Frontend

Современный SPA на Svelte 5

Почему Svelte 5?

Svelte 5 — это компиляторный фреймворк, который генерирует минимальный JavaScript без виртуального DOM. Новая система реактивности Runes ($state, $derived, $effect) даёт предсказуемый и производительный код.

  • Нет Virtual DOM — прямая манипуляция DOM, минимальный бандл
  • Runes — $state, $derived, $props вместо магических синтаксисов
  • Scoped CSS — стили автоматически изолированы в компонентах
  • SvelteKit — маршрутизация, layouts, SSG, обработка ошибок

Основные зависимости

06 · Стек технологий · Backend

Backend

ASP.NET Core 10 · Clean Architecture

Почему .NET 10?

ASP.NET Core — один из самых производительных серверных фреймворков. Строгая типизация C#, богатая экосистема, Entity Framework Core для ORM, встроенная поддержка WebSocket через SignalR.

  • Высокая производительность — Kestrel, async/await, пулы потоков
  • Type-safe — C# generics, nullable reference types
  • EF Core — Code-First миграции, Fluent API, LINQ
  • SignalR — встроенный WebSocket для реальных уведомлений
  • FluentValidation — декларативная валидация DTO

Security

Аутентификация и авторизация

JWT + Refresh Token

Двухуровневая система токенов с безопасным хранением в HttpOnly-cookies.

JWT Access30 минут, Bearer + Cookie (HttpOnly, Secure, SameSite)
Refresh Token7 дней, SHA256-хеш в БД, prefix «sk-», Cookie
JWT Claimsjti, sub, role, email, avatar, profile id
Key RotationПоддержка текущего + предыдущего ключа
JWT BlacklistRedis: jwt_blacklist:{jti}, TTL = остаток жизни

Сессии

Полноценное управление сессиями с детальным трекингом.

  • UserSession — 10 дней жизни
  • Трекинг: устройство, User-Agent, IP, город
  • Отзыв сессии с указанием причины
  • Просмотр активных сессий в настройках
  • При смене пароля — отзыв всех сессий
  • При удалении аккаунта — очистка всех токенов

Двухфакторная аутентификация (TOTP)

Полноценная реализация TOTP (RFC 6238) для второго фактора аутентификации.

  • Генерация 20-байтового секрета (Base32)
  • HMAC-SHA1, 30-секундное окно, 6 цифр
  • Допуск ±1 окно для компенсации расхождения часов
  • Шифрование секрета: AES-256-CBC
  • QR-код для настройки в Authenticator-приложении

Поток 2FA

1. Пользователь вводит логин/пароль ✓
2. Сервер проверяет, включён ли 2FA
3. Если да → Redis: pending challenge (5 мин TTL)
4. Клиент запрашивает 6-значный код
5. Сервер верифицирует код → выдаёт JWT + Refresh

Docker best practices

  • Multi-stage builds — frontend (2 стадии), backend (3 стадии)
  • Non-root — nginx-unprivileged, ASP.NET $APP_UID
  • Read-only FS — frontend с tmpfs для /var/cache
  • Health checks — на каждом сервисе
  • Resource limits — memory limits на каждый контейнер
  • Dependency ordering — depends_on с conditions
  • Alpine images — минимальный размер образов

10 · Авторские решения

Наши решения

Авторские решения и аргументация

Ряд функций был оставлен на усмотрение разработчика. Ниже — наши решения в зонах неопределённости задания и аргументация выбора.

1. Верификация компаний через DaData API

Решение: Работодатель указывает ИНН при регистрации. Система автоматически проверяет его через DaData API (база ФНС РФ), извлекает наименование, адрес и статус юрлица. Куратор видит результат проверки и принимает решение.

Аргументация: ИНН — универсальный идентификатор российских юрлиц, проверка через ФНС даёт достоверные данные. Автоматизация снижает нагрузку на куратора, при этом финальное решение остаётся за человеком. Бейдж «Верифицирована» повышает доверие соискателей к вакансиям.

2. Модерация контента — превентивный подход

Решение: Вакансии и мероприятия публикуются после проверки куратором (pre-moderation). Куратор видит превью карточки с тегами и зарплатой, может одобрить или отклонить с комментарием. Аудит-лог фиксирует все действия.

Аргументация: Для образовательной платформы, ориентированной на студентов, важно исключить мошеннические вакансии и неуместный контент до его публикации. Все действия модератора логируются для подотчётности.

3. Нетворкинг-система

Решение: Профессиональные контакты с двусторонним подтверждением. Контакты видят интересы друг друга и могут рекомендовать на вакансии. Три уровня приватности: скрыть отклики, скрыть резюме, полностью приватный профиль.

Аргументация: Студентам важно не только найти работу, но и построить профессиональные связи. Рекомендации от коллег — один из самых эффективных способов трудоустройства. Гибкая приватность учитывает, что не все готовы открыто искать работу.

4. Система тегов — crowdsourced

Решение: Стартовый набор тегов по IT-специализациям (языки, фреймворки, роли). Работодатели могут предлагать новые теги, кураторы — одобрять их и управлять списком. Теги отсортированы по популярности.
Аргументация: IT-стек постоянно меняется, crowdsourced-модель позволяет платформе оставаться актуальной без ручного обновления каталога технологий.

10 · Авторские решения · Продолжение

Дополнительные решения

Авторские решения (продолжение)

5. Модульная карточка на карте

Решение: При наведении на маркер — popup с названием, компанией, зарплатой, ключевыми навыками и ссылкой «Подробнее».

Почему: Баланс информативности и компактности. Пользователь может оценить вакансию, не покидая карту.

6. Избранное для неавторизованных

Решение: localStorage для гостей, серверная синхронизация после регистрации. Избранные компании выделяются на карте акцентным цветом.

Почему: Снижает барьер входа — можно начать пользоваться без регистрации.

7. Markdown для описаний

Решение: Редактор Markdown с тулбаром (жирный, курсив, списки, ссылки, код) и live-preview. Рендеринг с DOMPurify-санитизацией.

Почему: Работодатели привыкли к Markdown (GitHub/GitLab). Форматирование делает описания вакансий структурированными и читаемыми, а санитизация защищает от XSS.

8. Профиль компании — медиаконтент

Решение: Фото-галерея офиса с lightbox-модалкой, видео-презентация, динамический список соцсетей.

Почему: Визуальный контент критически важен для привлечения молодых специалистов. Фото офиса и корпоративные видео помогают сформировать представление о компании.

9. Real-time уведомления

Решение: Тройная доставка: SignalR (в приложении) + Web Push (VAPID) + БД (persistent). 8 типов событий.

Почему: Студенты не сидят на платформе постоянно. Push-уведомления гарантируют, что важные обновления (приглашение на собеседование) не будут пропущены.

10. Command Palette (Ctrl+K)

Решение: Глобальный поиск по Ctrl+K: вакансии, компании, мероприятия, страницы. Debounced запросы, навигация клавиатурой, история поиска.

Почему: Паттерн, знакомый IT-аудитории (VS Code, GitHub, Notion). Ускоряет навигацию для опытных пользователей.

11. Сравнение вакансий

Решение: Возможность добавить до 3 вакансий в сравнение. Фиксированная панель внизу, side-by-side таблица: зарплата, формат, навыки, местоположение.
Почему: Помогает принять обоснованное решение при выборе между несколькими стажировками или вакансиями — частая ситуация для начинающего специалиста.
